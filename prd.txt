PRD: Pip — The Desktop Quota Burn Pet (Windows)
Status: Draft  
Author: You  
Date: 2025-06-14  
Platform: Windows 10/11  
Scope: Single-file executable desktop widget
1. Elevator Pitch
A small, borderless desktop pet that floats above all windows and visualizes how much of your weekly Claude (or generic AI service) token/quota budget you have burned. The pet becomes happy, excited, and celebratory the closer you are to 100% utilization, and becomes sad, grumpy, or "mad" when you are under-utilizing your subscription. It persists its position, reads local session data (or manual input) for usage, and is designed to run quietly in the background via the system tray.
2. Goals
1. Always visible but never in the way — compact, borderless, click-through-capable when idle, draggable when grabbed.
2. Emotionally reactive — the pet's face/animation state maps directly to real-time burn percentage.
3. Zero-config startup — runs from a single command; optional config file for preferences.
4. Privacy-first usage tracking — reads from a local JSON file/session store rather than requiring cloud API keys (optional API polling as a stretch feature).
5. Lightweight & dependency-free — built with Python standard library only.
3. Non-Goals
- Not a full system monitor or task manager.
- Not a browser extension or Electron app (intentionally lightweight).
- Not required to authenticate with Anthropic/Claude API in the base version.
- Not cross-platform (Windows only to keep scope tight; other agents may adapt).
4. Functional Requirements
4.1 Core Window Behavior
ID	Requirement
F1	Window is borderless (no title bar, no OS chrome).
F2	Window is always on top of all other application windows.
F3	Window background is transparent/click-through except for the pet body.
F4	User can drag the pet by clicking and holding on the pet body to reposition.
F5	Window position is persisted across restarts (read/write config.json).
F6	Pet sits in the system tray on startup; right-click tray icon shows "Show / Hide / Exit".
F7	Double-clicking the pet opens a settings popup (see §4.3).
4.2 Emotion / State Engine
Burn percentage is calculated as:
burn_pct = (tokens_used_this_week / weekly_quota) * 100
The pet must visually reflect one of five states:
Burn %	Mood Name	Visual (Emoji fallback)
0–10%	Furious / Mad	😠 / 😡
11–30%	Sad / Disappointed	😢 / 🥺
31–65%	Neutral / Bored	😐
66–99%	Happy / Excited	😊 / 🤩
100%+	Overjoyed / Celebrating	🥳 / 😆
- Transitions: Mood changes should animate/tween over 300–500 ms (fade or scale pop) rather than snapping instantly.
- Idle Animation: Even in a static mood, the pet should "breathe" (scale 1.0 → 1.03 → 1.0 over 2 s) so it feels alive.
4.3 Settings / Data Popup
A small modal (or attached panel) that appears on double-click:
Field	Type
Weekly Quota (tokens)	Number input
Tokens Used This Week	Number input
Data Source	Dropdown
Poll Interval (sec)	Number input
Data Source Options:
1. Manual — user types tokens used directly.
2. Local JSON File Watch — app watches a usage.json file on disk for updates (schema in §5.2).
3. Claude API (Stretch) — poll Anthropic usage endpoint with a user-supplied API key.
When the popup is closed, values are saved to config.json and the pet immediately updates its mood.
4.4 Data Ingestion (Local Session Mode)
If Data Source = Local JSON File Watch, the app monitors <app_dir>/usage.json and hot-reloads on change.
{
  "week_start": "2025-06-09",
  "tokens_used": 420000,
  "quota": 1000000
}
If week_start mismatches current ISO week, the app resets tokens_used to 0 internally and writes back.

Language	Python 3.10+
GUI	Tkinter + tkinter.ttk
System Tray	tkinter + win32 via ctypes OR pure tkinter.Tcl tray stub

6. UI/UX Specifications
6.1 Dimensions & Layout
- Base size: 220 × 260 px (scalable via pet_scale).
- Shape: Rounded rectangle or circular "orb" body (canvas arc) to feel friendly.
- Z-Order: Always HWND_TOPMOST.
6.2 Visual Elements
1. Pet Face: Centered, ~60% of body height. Rendered either via:
- Canvas-drawn vector face (circles for eyes, bezier curves for mouth), OR
- Large emoji label as a pragmatic fallback.
2. Burn Bar: Horizontal bar beneath the face (width 70%, height 8 px, rounded caps). Fills left-to-right; color matches mood tint.
3. Percentage Badge: Small rounded pill label above or below the bar showing "42%". Hidden if show_percentage_badge is false.
4. Tooltip: On hover, show tooltip: 420K / 1M tokens used this week.
6.3 Drag Behavior
- ButtonPress-1 on pet body records drag_start_x, drag_start_y.
- B1-Motion moves window via geometry() deltas.
- ButtonRelease-1 saves new x, y to config.json.
6.4 Context Menu
Right-click on the pet body opens a tiny context menu:
- Settings… → opens popup
- Reset Position → centers on screen
- Quit → exits
7. State Machine Logic (Pseudocode)
def compute_mood(burn_pct):
    if burn_pct <= 10:  return Mood.FURIOUS
    if burn_pct <= 30:  return Mood.SAD
    if burn_pct <= 65:  return Mood.NEUTRAL
    if burn_pct < 100:  return Mood.HAPPY
    return Mood.OVERJOYED

def update_loop():
    usage = load_usage()
    pct = min(usage.tokens / usage.quota, 1.25)  # cap visual at 125%
    mood = compute_mood(pct)
    render(mood, pct)
    schedule(update_loop, interval=1_000)  # 1 second heartbeat
8. Stretch Goals (Optional)
1. Claude API Polling — real endpoint integration with API key input.
2. Sound Effects — tiny WAVs on mood transition (Windows winsound module works well).
3. Notification Toast — Windows native toast when hitting 50/75/100%.
4. Auto-updater — check GitHub releases on startup.
5. Themes — dark/light body tint variants.
9. Acceptance Criteria
- python pip.py opens immediately without pip-installing anything (standard lib only).
- Pet is always visible above maximized Chrome/VS Code windows.
- Dragging the pet to the bottom-right corner, closing, and reopening restores it to that exact position.
- Changing the "Tokens Used" slider from 0 to 1,000,000 causes the face to march through all five moods smoothly.
- Closing via system tray exits cleanly (no ghost Python process).
- usage.json dropped into the folder auto-updates the pet within the configured poll interval.
